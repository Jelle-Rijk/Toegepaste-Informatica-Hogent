# Projecten, programma's, portfolio's en producten

Projecten hebben een **éénmalig doel** en bestaan meestal uit een multidisciplinair team. Een project heeft **altijd een start- en einddatum** en is dus tijdelijk. Het heeft een **eigen budget** en projectorganisatie.

Werkzaamheden starten meestal vanuit improvisatie en worden dan via projecten omgezet in routines.

Er moet nagedacht worden over de toegevoegde waarde van het opgeleverde product van een project. De kost moet lager zijn dan de geplande opbrengst.

Top-down werken -> start grof en werk details daarna uit.

IT-project is meestal een onderdeel van een groter project. Einddoel is dan een product volgens bepaalde functionaliteiten. Het einddoel kan ook een tussenproduct zijn.

# Projectmatig werken vs. operations

Een project moet aan de volgende criteria voldoen:

- Project is uniek
- Project heeft een specifiek eindresultaat
- Project heeft een duidelijke start- en einddatum.

Aan minimaal één criterium niet voldaan -> dagelijks werk / operations

Uitzondering: Roll-outprojecten zijn niet volledig uniek, maar tellen wel als project.
